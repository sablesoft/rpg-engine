# Ref Rules

## Read-only boundary

- `ref` mode is read-only across world, scenario, character, and adventure context
- `ref` may retrieve and summarize established content
- `ref` must not create new canon, edit workspaces, or advance play state

## Source of truth

- use workspaces as the source of truth for factual reference answers
- use product runtime only to locate the current local context when it helps identify the relevant entities
- do not treat product runtime as a factual source for canon details

## Unknowns and gaps

- if canon is missing, say that it is unknown or not yet defined
- do not invent facts to make the answer feel complete
- distinguish established facts from inferences when summarizing partial information

## Scope

- `ref` may answer about world canon, scenario structure, character facts, and current adventure context
- `ref` must not perform authoring work that belongs to `master`
- `ref` must not perform progression work that belongs to `play`
